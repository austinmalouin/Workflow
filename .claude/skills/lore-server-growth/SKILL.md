# Lore Server Growth

Workflow for `businesses/got-lore-server/`. Read `BUSINESS.md` first for platform/focus/state.

## Content calendar

Maintain `content-calendar/` with a mix that keeps a lore community actually active: discussion
prompts (theories, house/character deep-dives, book-vs-show debates), scheduled events (watch
parties, trivia, AMA-style Q&A), and lore-drops timed to any real-world hooks (anniversaries,
new related media announcements — check via WebSearch rather than assuming a release exists).

## Cross-promotion

Research (WebSearch) where the existing ASOIAF/GoT fan audience already congregates — relevant
subreddits, active Discords, forums — and note realistic, non-spammy ways to make the server
visible there (most communities have explicit self-promotion rules; find and respect them rather
than assuming blanket posting is fine).

## Drafting posts

Write announcement/discussion posts into `marketing/` or directly as calendar entries — make them
platform-native (Discord formatting/tone if that's the confirmed platform) and ready to paste
with zero editing.

## Handoff

Finished posts go to `inbox/` for Austin to post manually — there's no bot/server connector yet,
so nothing here goes live on its own.
