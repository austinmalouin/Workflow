# Etsy Ops

Workflow for `businesses/etsy-shop/` (SteadyLedgerPrints). Read `BUSINESS.md` there first for
niche/goals/state — then, if you need anything current (listings, prices, sales), check the live
shop yourself via the browser tools rather than treating `BUSINESS.md` as always up to date or
asking Austin to re-report it.

## Listing creation

1. Keyword research first, always — WebSearch for real current search-demand signals (Etsy's own
   suggest/trending terms, third-party keyword tools' public data, competitor listing titles that
   are ranking) before writing a single title. Don't guess keywords from general knowledge.
2. Draft title, 13 tags, and description following Etsy's actual format constraints (title length
   limits, tag count/character limits) — verify these limits via web search rather than assuming,
   since Etsy changes them periodically.
3. Pricing: check 5-10 comparable active listings for the same/similar product before proposing a
   price — factor in Etsy fees (listing, transaction, payment processing — look up current rates)
   plus materials/time if known.
4. Save the draft to `listings/<product-slug>.md` with: title, tags, description, price + fee
   breakdown, and the comparables used to justify the price.

## Seasonal/campaign planning

Maintain a rolling view in `marketing/seasonal-calendar.md` of upcoming shopping seasons relevant
to the niche (holidays, back-to-school, wedding season, etc. — whatever actually applies) with
lead time for listing new seasonal products before demand peaks, not after.

## Before handing off to inbox/

Every draft going to `inbox/` should be complete enough that Austin can paste it directly into
Etsy with zero editing needed — no placeholder brackets, no "insert X here."

## Publishing (after approval only)

Drafting is unrestricted; publishing/editing the live shop is external-facing and always needs
Austin's explicit go-ahead first (route through the `approval-queue` skill, or get direct
confirmation in chat for a specific change). Once approved, use the browser tools to actually
enter the change in Shop Manager yourself rather than making Austin paste it in — that's the
point of having live shop access. Take a screenshot after publishing to confirm it landed as
intended.
