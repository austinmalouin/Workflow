# Minecraft Content

Workflow for `businesses/minecraft-channel/`. Read `BUSINESS.md` first for format/niche/cadence.

## Idea generation

Ground every idea in real current demand — WebSearch what's trending in Minecraft content right
now (new updates, popular modpacks/challenges, seed types, redstone trends), not evergreen
guesses. Note the source/reasoning next to each idea in `content-calendar/`.

## Scripting

1. Write to `scripts/<episode-slug>.md`: hook (first 10-15 seconds — this is what determines
   click-through, write it deliberately), outline/beats, estimated runtime, any commentary
   talking points if the format is voiced.
2. Match structure to format — a Let's Play needs different pacing notes than a Short or a
   tutorial.

## Thumbnail + metadata

For each episode, draft: title (test 2-3 options with reasoning), description (with any
timestamps/keywords), tags, and a thumbnail brief (composition, text overlay, focal point) —
brief Canva via `ToolSearch` if generating the actual image.

## Upload package

Bundle title + description + tags + thumbnail brief + script into a single file in `inbox/`
per episode — this is the handoff point where Austin's own recording/editing/uploading takes
over, since there's no upload connector yet.

## Cadence discipline

Keep `content-calendar/` realistic against what's actually achievable given Austin has to record
and edit manually right now — an overcommitted schedule that quietly slips isn't better than an
honest, sustainable one.
